# v1.1.0 작업 계획

## 목적

v1.1.0은 지도 장애 대응 이후, 사용자가 지도에 의존하지 않고도 맛집 목록에서 지역, 지원금 사용 가능 여부, 검색 조건을 기반으로 Store를 탐색할 수 있도록 개선한 버전이다.

이번 버전의 목적은 지도 실패 상황에서도 사용자가 서비스를 계속 이용할 수 있도록 `/stores` 탐색 경험을 강화하는 것이다. Store 데이터에는 실사용 정보인 `고유가 피해지원금 사용 가능 여부`를 추가하고, 현재 등록 가능한 전국 주소 범위와 목록 탐색 필터 범위를 맞춘다. 또한 Store 목록, StoreBox, 맛집 등록, 수정, 찜, 댓글 등 주요 Store 관련 UI/UX를 v1.1.0 기준으로 안정화한다.

## 배경

최근 작업에서는 Kakao Map SDK 실패와 Store API/DB 실패 상태를 분리하고, 지도 실패 시 10개 맛집 미리보기 fallback을 제거했다. 대신 지도 실패 카드에서 `/stores`로 이동할 수 있는 CTA를 제공하여, 지도 로드 실패 상황에서도 사용자가 맛집 목록으로 이동할 수 있게 했다.

이 흐름 이후에는 `/stores`에서 제공하는 정보와 탐색 UX가 중요해진다. 지도에 의존하지 않는 탐색이 가능하려면 목록에서 지역, 검색어, 지원금 사용 가능 여부 같은 조건을 안정적으로 사용할 수 있어야 한다.

현재 맛집 등록은 Daum Postcode 도로명 주소 검색을 통해 전국 주소 등록이 가능하고, 등록된 Store는 메인 맵 마커에도 표시된다. 반면 지역 검색 필터는 서울/춘천 중심 구조에 머물러 있어, 등록 가능한 주소 범위와 탐색 가능한 필터 범위가 일치하지 않는다. v1.1.0에서는 이 불일치를 줄이기 위해 지역 검색 필터를 전국 단위로 확장할 예정이다.

목적:

- 기존 등록, 수정, 조회, API, type 흐름에서 새 필드가 일관되게 전달되도록 한다.

작업 범위:

- 고유가 피해지원금 사용 가능 필드 추가
- 지역 검색 필터를 전국 단위로 확장
- 현재 서울/춘천 중심 필터 구조 확인
- Daum Postcode 도로명 주소 검색으로 등록 가능한 전국 주소와 목록 필터 범위를 맞추기
- 기존 등록/수정/조회/API/type 흐름에 새 필드 반영
- 전국 시/도 필터 검증용 개발 fixture 준비
- 개발 DB 검증용 fixture import 스크립트 준비

예상 필드명:

- `acceptsPaySupport`

검증 기준:

- Prisma Store 모델과 migration에 `acceptsPaySupport` 필드가 반영되어 있다.
- Store 생성, 수정, 조회 API에서 새 필드가 정상적으로 처리된다.
- 기존 Store 데이터 조회 시 오류가 발생하지 않는다.
- 맛집 등록/수정 화면에서 지원금 사용 가능 여부를 설정할 수 있다.
- 지역 필터가 서울/춘천 외 전국 주소 Store 탐색에도 사용할 수 있는 구조로 확장되어 있다.
- 전국 시/도 대표 지역 기준 더미 Store 데이터가 개발용 fixture로 준비되어 있다.
- 개발 DB에 fixture를 반영한 뒤 전체 목록, 시/도 필터, 메인 지도 마커 렌더링을 확인할 수 있다.
- 메인 지도 마커 표시 흐름에 회귀가 없다.
- `yarn lint`, `yarn build`가 통과한다.

추천 커밋 메시지:

### PR #3. 지원금 필터 및 배지 고도화

목적:

- 사용자가 지원금 사용 가능 Store를 목록과 상세 화면에서 쉽게 확인할 수 있도록 배지와 안내 문구를 추가한다.
- 지역 필터, 검색어 필터와 함께 지원금 사용 가능 매장만 보기 필터가 동작하도록 한다.

작업 범위:

- Store 목록 카드에 지원금 사용 가능 배지 표시
- Store 상세 페이지에 지원금 사용 가능 안내 표시
- 지원금 사용 가능 매장만 보기 필터 추가
- 지원금 필터 상태를 API query / React Query key / Zustand 상태 중 현재 구조에 맞게 반영
- 지역 필터, 검색어 필터와 함께 동작하도록 확인
- 빈 결과 UI 문구 추가
- 태극마크/정부상징 대신 체크, 원화, 카드 등 중립 아이콘 검토
- 유의 문구 추가

추천 문구:

- 목록 카드: `지원금 사용 가능`
- 상세 페이지: `고유가 피해지원금 사용 가능`
- 유의 문구: `사용 가능 여부는 카드사, 지역, 결제수단, 가맹점 상태에 따라 달라질 수 있습니다.`

사용하지 않을 표현:

- 태극마크, 정부상징, 공식 인증처럼 보이는 표현
- `정부 인증 가맹점`
- `행정안전부 인증 매장`
- `공식 사용처 인증`

검증 기준:

- 지원금 사용 가능 Store에는 목록 카드 배지가 표시된다.
- 지원금 사용 불가 Store에는 배지가 표시되지 않는다.
- Store 상세 페이지에서 지원금 사용 가능 안내와 유의 문구를 확인할 수 있다.
- 지원금 사용 가능 매장만 보기 필터가 단독으로 동작한다.
- 지원금 필터가 지역 필터, 검색어 필터와 함께 동작한다.
- 지원금 필터 결과가 없을 때 빈 결과 UI 문구가 표시된다.
- 정부 인증 또는 공식 인증처럼 보이는 UI, 문구, 아이콘을 사용하지 않는다.
- `yarn lint`, `yarn build`가 통과한다.

추천 커밋 메시지:

- `feat(store): add fuel support filter and badges`

## Known Issues

- Kakao Map SDK 또는 외부 API 실패 상태는 네트워크/브라우저 환경에 따라 재현성이 달라질 수 있다.
- 지원금 사용 가능 여부는 서비스 내부 참고 정보이며, 실제 사용 가능 여부는 카드사, 지역, 결제수단, 가맹점 상태에 따라 달라질 수 있다.
- 개발 fixture는 전국 시/도 필터 검증용 데이터이므로 운영 데이터 품질을 보장하지 않는다.
- 모바일 Lighthouse는 수동 1회 측정만으로 판단하지 않고, 후속 반복 측정 기준을 별도로 마련한다.
