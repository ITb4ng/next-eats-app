## React Query focus refetch 정리

증상:

- 다른 탭이나 다른 프로그램을 본 뒤 서비스 화면으로 돌아오면 Store 목록/상세 화면에서 로더가 다시 노출됐다.
- 사용자는 새로고침처럼 느낄 수 있었다.

원인:

- React Query의 기본값인 `refetchOnWindowFocus: true`로 인해 포커스 복귀 시 stale query가 다시 fetch됐다.
- 일부 화면에서 `isFetching`을 초기 로딩처럼 사용해, 기존 데이터가 있는데도 전체 로더가 노출됐다.

반영:

- `src/pages/_app.tsx`
  - QueryClient 기본 옵션에 `refetchOnWindowFocus: false` 추가.
- `src/pages/stores/index.tsx`
  - 목록 하단 로더를 `isFetchingNextPage` 기준으로 변경.
- `src/pages/stores/[id]/index.tsx`
  - 상세 전체 로더 기준을 `isLoading`으로 변경.

판단:

- Store 목록/상세는 초 단위 최신성이 중요한 화면이 아니므로, focus 복귀 자동 refetch보다 읽던 흐름 유지가 더 중요하다고 판단했다.
- 이후 실시간성이 필요한 화면이 생기면 해당 query 단위에서만 `refetchOnWindowFocus: true`를 명시하는 방식이 적합하다.

## 지역명 alias 보강

기존 규칙:

- `강원특별자치도` <-> `강원도`
- `전북특별자치도` <-> `전라북도`

추가 규칙:

- `제주특별자치도` <-> `제주도`

반영:

- `src/data/region.ts`
  - `REGION_NAME_ALIASES`에 제주 alias 추가.
  - `REGION_CANONICAL_NAMES`에 `제주도 -> 제주특별자치도` 추가.

확인:

```txt
getCanonicalRegionName("제주도")
-> 제주특별자치도

getDistrictSearchCandidates("제주특별자치도 제주시")
-> ["제주특별자치도 제주시", "제주도 제주시"]

getDistrictSearchCandidates("제주도 제주시")
-> ["제주도 제주시", "제주특별자치도 제주시"]
```
