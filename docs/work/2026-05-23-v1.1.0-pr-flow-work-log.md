## 요약

2026-05-23 작업은 v1.1.0 PR 흐름을 문서화하고, PR #1 범위인 Store 지원금 사용 정보 추가와 전국 지역 필터 기반 확장을 진행한 작업이다.

v1.1.0의 핵심 방향은 지도 장애 대응 이후 사용자가 지도에 의존하지 않고도 `/stores` 목록에서 지역, 검색 조건, 지원금 사용 가능 여부를 기반으로 Store를 탐색할 수 있게 만드는 것이다. 이번 작업에서는 그 기반으로 Store 데이터 모델에 `acceptsPaySupport`를 추가하고, 서울/춘천 중심이던 지역 필터를 전국 시/도 단위로 확장했다.

## PR #1 작업 상세

### 1. 지원금 사용 가능 필드 추가

Store 모델에 `acceptsPaySupport Boolean @default(false)` 필드를 추가했다.

반영 범위:

- Prisma Store 모델
- Prisma migration
- Prisma Client generate
- Store 생성 API
- Store 수정 API
- Store 조회 API
- Store type/interface
- 맛집 등록 폼
- 맛집 수정 폼

확인 결과:

- 신규 Store 등록 시 체크박스 값이 DB에 저장된다.
- 기존 Store는 기본값 `false`로 안전하게 조회된다.
- 로그인 후 Store 수정 시 `acceptsPaySupport` 값을 변경할 수 있다.
- Prisma Studio에서 `true`/`false` 값이 정상 반영되는 것을 확인했다.

### 2. DB schema 불일치 수정

`acceptsPaySupport` 필드가 Prisma schema에는 존재하지만 실제 DB에는 컬럼이 없어 `/api/stores`에서 P2022 에러가 발생했다.

발생 에러:

- `The column Store.acceptsPaySupport does not exist in the current database.`

수정 내용:

- migration `20260523133526_add_accepts_pay_support_to_store` 적용
- SQL 기준:

```sql
ALTER TABLE "Store" ADD COLUMN "acceptsPaySupport" BOOLEAN NOT NULL DEFAULT false;
```

확인 결과:

- `npx prisma migrate deploy` 성공
- `npx prisma migrate status`에서 `Database schema is up to date!` 확인
- `/api/stores` 200 응답 확인
- 메인 페이지의 `store-api-error / status 500` 상태가 사라짐

### 3. 지역 필터 전국 시/도 단위 확장

기존 지역 필터는 서울특별시와 강원도 춘천 중심 구조였다. PR #1에서는 최소 기준으로 전국 시/도 옵션을 추가하고, 시/도만 선택해도 `address contains` 기반으로 목록 필터가 동작하도록 확장했다.

반영한 시/도:

- 서울특별시
- 부산광역시
- 대구광역시
- 인천광역시
- 광주광역시
- 대전광역시
- 울산광역시
- 세종특별자치시
- 경기도
- 강원특별자치도
- 충청북도
- 충청남도
- 전북특별자치도
- 전라남도
- 경상북도
- 경상남도
- 제주특별자치도

호환 처리:

- `강원특별자치도` 검색 시 기존 `강원도` 주소도 함께 검색
- `전북특별자치도` 검색 시 기존 `전라북도` 주소도 함께 검색
- URL query에 기존 지역명이 들어온 경우 UI에서는 표준 지역명으로 복원

### 4. 전국 시/도 검증용 fixture와 import 스크립트

전국 시/도 필터와 메인 지도 마커 흐름을 검증하기 위해 개발용 fixture를 추가했다.

추가 파일:

- `prisma/fixtures/v1.1.0-region-filter-stores.json`
- `scripts/seed-v1.1.0-region-fixtures.ts`

package script:

```bash
yarn seed:v1.1.0:region-fixtures
```

fixture 기준:

- 전국 17개 시/도별 대표 지역 Store 1개씩
- 각 데이터에 `name`, `category`, `address`, `lat`, `lng`, `acceptsPaySupport`, `foodCertifyName`, `storeType`, `phone` 포함
- `acceptsPaySupport`는 `true` 9개, `false` 8개로 혼합
- `name + address` 기준으로 중복 생성 없이 update 가능

실행 결과:

```txt
created=17, updated=0, total=17
created=0, updated=17, total=17
```

### 5. 검증 결과

API 검증:

- `/api/stores` 전체 응답에 fixture 17개 포함 확인
- 17개 시/도 필터 각각에서 기대 Store 노출 확인
- 기존 Store 데이터 조회 오류 없음

브라우저 검증:

- `/stores` 전체 목록에서 fixture Store 노출 확인
- `/stores?district=...` 기준 17개 시/도별 Store 노출 확인
- 메인 `/`에서 `store-api-error` 미노출 확인
- 메인 API 응답 기준 lat/lng가 있는 Store 36개 중 fixture 17개 포함 확인
- 메인 지도 축소 후 marker image 36개 렌더링 확인
- 렌더링 가능 Store 수와 marker image 수 일치 확인
