## 요약

v1.1.0 PR 3 작업으로 PR 1에서 추가된 `acceptsPaySupport` 값을 실제 탐색 조건과 UI 표시로 연결했다. Store 목록, 상세 페이지, StoreBox에서 지원금 사용 가능 Store를 중립적인 정보성 배지로 표시하고, `/stores` 목록에는 지원금 사용 가능 매장만 보기 필터를 추가했다.

작업 후 모바일 목록 배지 폭, StoreBox 아이콘 정렬, 상세 페이지 지도 높이와 댓글 노출 흐름도 함께 정리했다. 이 보조 조정은 지원금 배지 추가로 생긴 UI 흐름과 상세 페이지 정보 위계를 안정화하는 범위이므로 v1.1.0 PR 3에 포함해도 적합하다고 판단했다.

## 반영 내용

### 지원금 배지

- 공통 `PaySupportBadge` 컴포넌트를 추가했다.
- Store 목록 카드에는 `지원금 사용 가능` 배지를 표시한다.
- Store 상세 페이지에는 `고유가 피해지원금 사용 가능` 안내와 유의 문구를 표시한다.
- StoreBox에는 공간을 해치지 않는 짧은 `지원금 사용 가능` 배지만 표시한다.
- `acceptsPaySupport`가 `false`, `null`, `undefined`인 Store에는 지원금 배지를 표시하지 않는다.
- 태극마크, 정부상징, 공식 인증처럼 보이는 문구나 아이콘은 사용하지 않았다.

### 지원금 필터

- `/stores` 검색 필터 영역에 `지원금 사용 가능 매장만 보기` 체크박스를 추가했다.
- 필터 상태는 URL query `acceptsPaySupport=true`와 Zustand 상태에 함께 반영한다.
- React Query key에 지원금 필터 상태를 포함했다.
- `/api/stores`에서는 query 값이 정확히 `true`일 때만 `acceptsPaySupport: true` 조건을 추가한다.
- `acceptsPaySupport=false` 또는 미전달 시 기존 전체 조회 흐름을 유지한다.

### 상세 및 보조 UI 정리

- 모바일 목록 카드에서 지원금 배지가 카드 폭만큼 늘어나지 않도록 compact pill 형태로 조정했다.
- StoreBox에서 지원금 배지가 표시될 때 카테고리 아이콘이 위로 치우쳐 보이지 않도록 왼쪽 헤더 그룹을 수직 중앙 정렬했다.
- Store 상세 페이지 정보 목록에서 사용자가 직접 확인할 필요가 낮은 위도/경도 값을 숨겼다.
- 상세 페이지 지도는 메인 지도와 분리해 `variant="detail"` compact 높이를 사용하도록 했다.
- 지도 SDK 실패 상태에서도 상세 페이지 지도 영역이 과하게 커지지 않고, 바로 아래 댓글 섹션이 보이도록 조정했다.

## v1.1.0 PR 3 적합도

적합:

- 지원금 배지와 지원금 필터는 v1.1.0 PR 3의 핵심 목적과 직접 일치한다.
- API query 조건과 React Query key/Zustand 상태 반영은 지원금 필터가 실제 탐색 조건으로 동작하기 위한 필수 변경이다.
- 모바일 배지 폭 조정과 StoreBox 아이콘 정렬은 지원금 배지 추가로 발생한 UI 균형 문제를 해결하는 후속 다듬기이므로 PR 3에 포함하기 적합하다.
- 상세 페이지 위도/경도 미노출과 지도 compact 높이 조정은 상세 페이지에서 지원금 안내와 댓글까지의 정보 흐름을 정리하는 작은 UI 안정화로, PR 3 후반 보조 조정으로 포함 가능하다.

주의:

- Store 데이터 모델 변경, Prisma migration 추가, 전국 지역 필터 구조 재설계는 하지 않았다.
- 맛집 등록/수정 폼, 찜, 댓글 UI 자체는 다시 리디자인하지 않았다.
- 정부 인증처럼 보이는 표현이나 정부기관 로고/상징은 사용하지 않았다.
- Kakao SDK 성공 상태의 실제 지도 렌더링은 로컬 네트워크/SDK 실패 상태 때문에 직접 확인하지 못했다. 실패 상태와 컨테이너 높이 변경은 확인했다.
