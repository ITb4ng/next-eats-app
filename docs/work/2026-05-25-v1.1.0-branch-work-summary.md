## 1. `feature/v1.1.0-store-ux-polish`

작업 목적:

- PR #2 범위인 Store UI/UX 안정화 작업을 마무리하고, 작업 중 발견된 보조 이슈를 정리한다.

주요 작업:

- React Query focus 복귀 시 불필요한 전체 로더가 반복 노출되는 흐름을 정리했다.
- `QueryClient` 기본 옵션에 `refetchOnWindowFocus: false`를 적용했다.
- Store 목록 하단 로더 기준을 `isFetchingNextPage`로 조정했다.
- Store 상세 전체 로더 기준을 `isLoading`으로 조정했다.
- 제주 지역명 alias를 보강했다.
  - `제주특별자치도` <-> `제주도`
- 전화번호 포맷팅 규칙 정리를 PR #2 체크리스트에 포함했다.

작업 판단:

- Store 목록, 상세, 등록/수정, 찜, 댓글의 상태 UI 안정화 범위에 포함하기 적합하다.
- 지원금 배지와 지원금 필터는 PR #3 범위로 분리했다.

관련 기록:

- `docs/work/2026-05-24-v1.1.0-pr2-follow-up-work-log.md`

## 2. `feature/v1.1.0-support-filter-badge`

작업 목적:

- PR #3 범위인 지원금 사용 가능 정보 표시와 필터를 실제 탐색 흐름에 연결한다.

주요 작업:

- 공통 `PaySupportBadge` 컴포넌트를 추가했다.
- Store 목록 카드에 `지원금 사용 가능` 배지를 표시했다.
- Store 상세 페이지에 `고유가 피해지원금 사용 가능` 안내와 유의 문구를 추가했다.
- StoreBox에도 compact 지원금 배지를 적용했다.
- `/stores`에 `지원금 사용 가능 매장만 보기` 필터를 추가했다.
- 지원금 필터 상태를 URL query, React Query key, Zustand 상태와 연결했다.
- `/api/stores` query에서 `acceptsPaySupport=true`일 때만 지원금 가능 Store 조건을 적용했다.
- 모바일 목록 카드에서 배지가 과하게 늘어나지 않도록 compact pill 형태로 보정했다.
- StoreBox 아이콘 정렬과 상세 페이지 지도 compact 높이를 보정했다.
- Store 상세 페이지에서 위도/경도 값이 UI에 노출되지 않도록 정리했다.

검증:

- `yarn lint`
- `yarn build`
- 지원금 필터 단독 확인
- 지원금 + 지역 필터 조합 확인
- 지원금 + 검색어 필터 조합 확인
- 모바일 390px 목록 배지 확인
- 상세 페이지 지원금 안내와 유의 문구 확인
- 지도 SDK 실패 상태의 상세 지도 compact 높이와 댓글 노출 흐름 확인

작업 판단:

- 지원금 배지와 필터는 PR #3 목적과 직접 일치한다.
- 정부 인증처럼 보이는 표현, 태극마크, 정부기관 로고/상징은 사용하지 않았다.
- Store 데이터 모델 변경이나 Prisma migration은 PR #1 범위라 이 브랜치에서는 다루지 않았다.

관련 기록:

- `docs/work/2026-05-24-v1.1.0-pr3-support-filter-badge-work-log.md`

## 4. `release/v1.1.0`

작업 목적:

- dev 기준으로 v1.1.0 릴리즈 후보 브랜치를 만들고, main 머지 전 최종 릴리즈 디테일을 정리한다.
- Next.js 16으로 올리지 않고 Next.js 15 계열의 안정 패치 버전으로 업데이트한다.
- 프로젝트 구조와 Pages Router 유지 판단을 문서화한다.
- Dependabot/보안 알림 대응과 Supabase wake-up 자동화 같은 v1.2.x 운영 자동화 후보를 문서화한다.

주요 작업:

- `release/v1.1.0` 브랜치를 dev 기준으로 생성했다.
- `package.json` version을 `1.1.0`으로 변경했다.
- Next.js를 15 계열 안정 패치 버전으로 업데이트했다.
  - `next`: `15.3.8` -> `15.5.18`
- React RC 버전을 안정 버전으로 정렬했다.
  - `react`, `react-dom`: `19.2.6`
- Prisma 관련 패키지를 업데이트했다.
  - `@prisma/client`, `@prisma/adapter-pg`, `prisma`: `7.8.0`
- 주요 런타임 의존성을 업데이트했다.
  - `axios`, `next-auth`, `pg`, `react-hook-form`, `react-icons`, `react-select`, `react-toastify`, `zustand`
- React 19 타입에 맞춰 `src/hooks/useIntersectionObserver.ts`의 ref 타입을 보정했다.
- 사용하지 않는 의존성 `install`, `recoil`을 제거했다.
- `dev:all` 스크립트에 필요한 `concurrently`를 추가했다.
- 운영 의존성 audit 0건을 위해 `yarn resolutions`를 추가했다.
- 루트의 `README-release-issue.md`를 `docs/hotfix/release-issue.md`로 이동해 루트 문서 구조를 정리했다.
- README를 v1.1.0 기준으로 업데이트했다.
  - v1.1.0 업데이트 요약
  - 기술 스택
  - Pages Router 유지 판단
  - 프로젝트 구조
  - 실행 및 검증 명령
- `docs/release/v1.1.0-dependency-update.md`를 작성했다.
- `docs/architecture/page-router-assessment.md`를 작성하고 이후 v1.2.x 운영 자동화 로드맵까지 확장했다.

v1.2.x 후속 제안으로 추가한 내용:

- Dependabot/보안 알림 대응 자동화
- 현재 프로젝트 구조와 Pages Router 정책을 유지하는 보안 업데이트 점검 스크립트
- major 업데이트 자동 적용 금지와 별도 이슈 분리
- `lint`, `build`, `audit`, smoke test 자동 실행
- Supabase 유휴 상태 방지를 위한 scheduled wake-up 자동화
- `/api/ping`, `/api/stores?take=1` 같은 읽기 전용 health check 후보
- Supabase wake-up 실패 시 GitHub Actions 실패 상태로 기록
- 운영 로그 템플릿 후보
- 참고 문헌 및 참고 사이트 정리

검증:

- `npm run lint`
- `npm run build`
- `yarn audit --groups dependencies`
- 로컬 브라우저에서 `/` 확인
- 로컬 브라우저에서 `/stores` 확인

결과:

- lint 성공
- build 성공
- 운영 의존성 audit 0건
- `/`, `/stores` 정상 로드 확인

주의:

- `next lint`는 Next.js 15에서는 동작하지만 Next.js 16에서 제거될 예정이므로 v1.2.x에서 ESLint CLI 전환이 필요하다.
- `react-query v3`는 React 19 peer dependency 경고가 남는다. build는 통과했지만 장기적으로 `@tanstack/react-query` 전환을 검토한다.
- `resolutions`로 보안 패치 버전을 고정한 하위 패키지는 lint/build/audit 기준으로 검증했다.
- 현재 `release/v1.1.0` 변경사항은 아직 커밋 전 상태다.

## 6. 릴리즈 진행 방향

권장 순서:

   - base: `main`

- GitHub default branch가 현재 `dev`이므로 PR 생성 시 base branch가 자동으로 `dev`가 될 수 있다.
- PR 생성 화면에서 base가 반드시 `main`인지 확인한다.
